# 蓝牙按键（BLE 按键互联）— 设计文档

> 独立车机 App。把各种蓝牙按键/旋钮接成可编程遥控：每个设备的每个手势可绑定任意车辆控制/系统动作。
> 对标原车「场景工坊」，覆盖其控制项；多产品可维护扩展；BindKey 登录一次自动获取，按键数量不限。
>
> 本文件是项目的总设计与开发蓝图。后续在新对话里据此实现。
>
> **应用名**：蓝牙按键 ｜ **包名**：`com.selfclink.ble`

## 0. 需求增补（2026-06-29 用户确认）

1. **包名 `com.selfclink.ble`，应用名「蓝牙按键」**。
2. **不存账号凭据**：米家账号只用于「登录一次 → 选择要接入哪些设备 → 读取这些设备的 BindKey」，账号密码 / serviceToken **不落盘**。
3. **登录二选一自由切换**：① 账号密码；② 扫码登录（二维码）。
4. **BindKey 来源**：从选中的「已绑定设备」里读取（米家云设备详情 `beaconkey`）。
   - ⚠️ 待确认点：BindKey 是设备解密密钥，运行时每帧广播都要用。是否允许把它（加密）存本地以便车机重启后离线可用？倾向**加密存本地**（EncryptedSharedPreferences，视作设备配置而非账号凭据）。见 §4。
5. **共创（社区共建）**：除了内置适配，用户可**自己抓取**蓝牙设备的控制规则、**导入**为设备配置，并可**导出/分享**给他人。降低单人维护压力。配置走声明式 JSON（§3.1），无需改代码即可新增大量简单按键设备。

## 1. 目标与范围

- **独立软件**：与机器人项目（ZKVision）完全分离，单独包名、单独 App。
- **设备**：先支持米家 BLE Mesh 按键/旋钮（如 lemesh.remote.ts00），框架可扩展到其它产品。
- **能力**：
  1. 扫描并识别已适配的蓝牙按键设备；
  2. **自动获取 BindKey**（米家云登录拉取），选中需要的设备即记住、自动绑定；
  3. 每个设备支持**任意多个**按键/手势，每个手势绑定**任意多个**动作（不限数量）；
  4. 动作覆盖原车「场景工坊」支持的车辆控制项（HVAC/车身/座椅/灯光/驾驶模式/360/哨兵/DVR…）+ 系统动作；
  5. 前台服务常驻，按键按下即执行。
- **可维护性**：新增一种产品 = 实现一个适配器接口 + 注册，不改主流程。

## 2. 总体架构（分层，单向依赖）

```
ui  ──────────────► automation ──► vehicle (车控执行)
 │                      │           system  (系统动作执行)
 │                      ▼
 │                   rules (规则存储/引擎)
 ▼
ble (扫描 + 产品适配框架) ◄── cloud (米家云 BindKey 获取)
```

- **ble**：BLE 扫描、产品识别、手势事件解析（产品适配框架）。
- **cloud**：米家账号登录 + 设备/BindKey 拉取。
- **vehicle**：ecarx 车控封装 + 动作目录（从 ZKVision 的 EcarxCarManager / VehicleController 移植并扩展）。
- **system**：系统动作（启动 App / 媒体 / 音量 / 亮度…，可选）。
- **automation**：规则模型、存储、执行引擎（设备+手势 → 动作[]）。
- **service**：前台常驻服务，装配上述对象、保活扫描。
- **ui**：界面（见 §6）。

## 3. 产品适配框架（可扩展核心）

新增产品只实现接口并注册，主流程不变。

```java
interface ProductAdapter {
    String productId();                 // 稳定标识，如 "mijia_mesh_remote"
    String displayName();               // "米家蓝牙旋钮"
    boolean matches(ScanContext scan);  // 据广播(service UUID/产品ID/名前缀)判断是否本产品
    CredentialSpec credentialSpec();    // 需要的凭据：NONE / BINDKEY(32hex) / ...
    List<Gesture> gestures();           // 支持的手势集合（单击/双击/长按/旋转…）
    GestureEvent parse(byte[] adv, Credential cred);  // 解析一帧广播→手势事件(或 null)
}
```

- `ProductRegistry`：持有全部适配器（内置 code 适配器 + 声明式 Profile 适配器）；扫描到设备时遍历 `matches` 命中归类。
- **首发适配**：`MijiaMeshRemoteAdapter`（MiBeacon FE95 + AES-CCM 解密，移植 ZKVision 的 `MiBeacon.java`）。
- **预留**：其它米家 Mesh 按键（不同 objId 表）、标准 BLE HID 遥控、自定义广播按钮。
- **手势模型**：`Gesture{id, name}`；产品自定义（旋钮=单击/双击/长按；带旋转的=左旋/右旋/按下旋转…）。数量不限。

### 3.1 共创：声明式设备配置（可导入 / 导出 / 分享）

为支持社区共建，把「一种设备怎么解析手势」表达成**声明式 JSON Profile**，用户抓包后自己写、导入即用、可一键导出分享。`DeclarativeAdapter` 把 Profile 适配成 `ProductAdapter`，与内置 code 适配器同框架运行。

```jsonc
{
  "schema": 1,
  "productId": "mijia.knob.ts00",   // 全局唯一稳定标识
  "displayName": "米家蓝牙旋钮",
  "author": "zhangyan",              // 分享署名（可空）
  "match": {                          // 任一条件均可，全部命中才算本产品
    "serviceUuid": "fe95",           // 16/128 bit，可空
    "productIdLe": "0x5149",          // FE95 帧内 pid（LE），可空
    "namePrefix": "" ,                // 设备名前缀，可空
    "manufacturerId": null            // 厂商数据 company id，可空
  },
  "credential": "BINDKEY16",          // NONE | BINDKEY16（16 字节 BindKey）
  "decode": {
    "codec": "mibeacon",              // 内置解码器 id：mibeacon | advmatch
    // codec=mibeacon：加密旋钮，按 objId → 手势
    "gestureObjMap": { "0x560c": "single", "0x560d": "double", "0x560e": "long" }
    // codec=advmatch（明文按钮，无需 BindKey）：按字节/掩码命中
    // "source": "service:fe95" | "manufacturer:0x038f",
    // "rules": [ { "gesture": "single", "offset": 7, "mask": "ff", "equals": "01" } ]
  },
  "gestures": [
    { "id": "single", "name": "单击" },
    { "id": "double", "name": "双击" },
    { "id": "long",   "name": "长按" }
  ]
}
```

- **内置解码器（codec）**：`mibeacon`（加密，原生 AES-CCM，需 BindKey）、`advmatch`（明文广播按字节/掩码匹配，覆盖大量简单按钮，无需密钥）。新增解码器才需要写代码；多数简单设备靠 `advmatch` 纯配置即可，这是共创的关键。
- **存储**：内置 Profile 打包在 assets；用户导入的存 app 私有目录 `profiles/`。`ProfileStore` 负责加载、校验、去重（按 `productId`，用户版可覆盖内置版）。
- **导入**：选 `.json` / `.btkey` 文件或粘贴文本 → 校验 schema → 落盘 → 注册到 Registry。
- **导出 / 分享**：把某 Profile 导出成单文件（系统分享面板）。⚠️ 导出**不含** BindKey（个人密钥），只含解析规则，可安全公开分享。
- **校验**：导入时校验 schema、codec 合法、match 至少一条、hex 合法；非法拒绝并提示。

## 4. BindKey 获取（米家云，登录一次）

参考开源「Xiaomi Cloud Tokens Extractor」流程，纯客户端，**账号只用一次**：

1. **登录**（二选一，自由切换）：
   - **账号密码**：account.xiaomi.com 三步登录（serviceLoginAuth2 → 可能二次验证）→ 拿 `userId / ssecurity / serviceToken`。
   - **扫码登录**：拉米家登录二维码（`qr/qrcode` 长轮询）→ 用户用米家/小米 App 扫码确认 → 拿同样的票据。
2. **拉设备列表**：用票据调米家 API（`/home/device_list`）→ 每个设备含 `mac`、`did`、`model`、Mesh 设备的 **beaconkey(BindKey)**。
3. **选择**：用户勾选要接入的设备 → 读取其 BindKey，与本地扫描设备按 mac 匹配，自动建条目。
4. **用完即弃账号凭据**：拉完设备/BindKey 后，**丢弃 `serviceToken / ssecurity / 密码`**，不落盘（满足「凭据不存」）。
5. **BindKey 存储（待确认）**：BindKey 是运行时解密必需的设备密钥。默认方案＝**加密存本地**（EncryptedSharedPreferences），视作「设备配置」而非账号凭据，使车机重启后离线可用；否则每次重启都要重新登录。**等用户拍板**。
6. **兜底**：保留手动输入 BindKey（也可来自他人分享的个人导出，不进公共共创）。

> 区域：米家分服务器区（cn/sg/de…），登录后按账号默认区，必要时让用户选区。

## 5. 动作目录（对标「场景工坊」）

动作 = 元数据驱动，便于扩展与 UI 自动渲染：

```java
class ActionDef {
    String key;            // "hvac_ac_toggle"
    String category;       // 空调/车身/座椅/灯光/驾驶/安全/系统
    String name;           // "AC 开关"
    ParamSpec param;       // NONE / TOGGLE / LEVEL_CYCLE / STEP / ENUM(options) / VALUE
    boolean parkGuard;     // 是否非P档拦截
    // 执行：funcId/zone/值 或 系统动作 handler
}
```

**车辆控制（移植自实车验证清单，全集）**
- 空调：总电源/AC/AUTO/ECO/双区同步/内外循环/风量±/温度±(主副后)/前后除霜/出风口模式/风向/扫风/香氛(开关·档·类型)/后排空调/预约空调。
- 座椅：加热/通风/按摩(程序·力度)，主副后排；方向盘加热；座舱过热保护等设置项。
- 车身：中控锁(锁/解锁)/四门/四窗(位置)/后备箱/后视镜折叠/充电口盖/一键关/驻车舒享/手动门模式/360环视。
- 其它（场景工坊含、走专属接口）：哨兵模式、行车记录仪 DVR、驾驶模式、灯光/氛围灯。
- **安全约束**：非 P 档拦截 解锁/开门/开后备箱（读不到挡位按非 P 处理）。

**系统动作（可选，二期）**
- 启动指定 App / 打开页面、媒体播放控制、音量±、屏幕亮度、截屏等。

> 实现底座：ecarx `com.ecarx.xui.adaptapi.*`（boot classpath），需**平台签名 + `android.uid.system`**。哨兵/DVR 走各自接口（注意开源版不写品牌字，用通用包名/funcId）。

## 6. UI 设计（待确认 → 见对话）

低饱和、8px 网格、车载大触摸区。主要页面：

1. **主页**
   - 顶部：米家账号状态（已登录/未登录）。
   - 「我的按键设备」卡片列表：每张卡显示设备名/类型/连接(广播)状态/已绑手势数；点开进编排。
   - 「+ 添加设备」大按钮。
2. **添加设备向导**（分步）
   - ① 扫描列表（已识别产品高亮，未知设备灰显）。
   - ② 选中 → 若该产品需 BindKey：优先「从米家账号自动获取」（一键匹配），否则手动输入。
   - ③ 命名 + 保存 → 自动绑定。
3. **按键编排页**（每设备）
   - 列出该产品全部手势；每个手势下是「动作列表」（可加多个，不限数量，按顺序执行）。
   - 「+ 添加动作」→ 分类动作选择器（空调/车身/座椅/…）→ 选动作 + 参数。
4. **米家账号页**：登录/退出、区域选择、「刷新设备 BindKey」。
5. **设置**：扫描模式、开机自启、关于。

> 设计稿确认后再写界面代码。

## 7. 工程与约束

- 包名：`com.selfclink.ble`。应用名：蓝牙按键。
- minSdk 24 / targetSdk 33（与车机一致）。
- 平台签名：EMAILADDRESS=android@android.com, CN=Android, O=Android。
- `sharedUserId="android.uid.system"`（车控所需）。
- `compileOnly` ecarx jar（从 ZKVision 复用 `libs/ecarx-adaptapi.jar`）。
- 版本号从 >2 起，每次改动递增；APK 以版本号命名。
- 前台服务（`foregroundServiceType="connectedDevice"`）保活扫描；按键广播短促，绑定后用 `SCAN_MODE_LOW_LATENCY`。
